# Priyanshu Singh Karki — Frontend Developer Portfolio

A creative, fast, accessible, and beautifully animated frontend developer portfolio built completely without frameworks. This project emphasizes clean semantic HTML, deep CSS styling, and natural, performant motion.

## ✨ Features

* **Zero Frameworks**: Built entirely with pure HTML5 and CSS3.
* **Responsive Design**: Fluid typography (`clamp()`) and container-aware breakpoints ensure it looks great on any screen size.
* **Custom Animations**: Keyframes, transforms, and transitions configured for natural motion (e.g., drifting blob backgrounds, scroll cues, hover effects, and progressive fade-ins).
* **Modern CSS**: Utilizes Custom Properties (Variables), CSS Grid, and Flexbox for robust, scalable layouts.
* **Accessible**: Semantic markup, proper contrast ratios, and structural clarity.
* **Immersive UI**: Features a sleek dark theme with dynamic gradients, noise textures, and grid overlays.
* **Working Contact Form**: Integrated with Web3Forms for seamless message delivery without a backend.

## 🛠 Tech Stack

* **HTML5**: Semantic and structured markup.
* **CSS3**: Advanced styling, animations, and responsive layouts.
* **Fonts**: Space Grotesk, Inter, and JetBrains Mono (via Google Fonts).

## 📂 Portfolio Sections

1. **Hero**: Engaging introduction with a download resume CTA, social links, and animated background decor.
2. **About**: Description of my frontend philosophy, craft, and approach.
3. **Skills**: Core toolkit highlighting HTML5, CSS3, Grid/Flexbox, Accessibility, Figma, and Git.
4. **Services**: Core offerings such as Landing Pages, UI Development, and Website Optimization.
5. **Projects**: Selected showcase of shipped work, including *Nimbus*, *BharatMart*, *Atlas*, and *RoadReady*.
6. **Journey**: A visual timeline of my development experience and learning milestones.
7. **Contact**: A functional contact form and contact details (Email, Phone, Location).

## 🚀 Getting Started

Since this project is built entirely with static files, no build step, server, or package installation is required.

1. Clone the repository or download the source files.
2. Ensure the `assets/` directory (containing images, logos, and resume) is correctly placed relative to `index.html`.
3. Open `index.html` in your favorite modern web browser.
4. Explore the interface!
